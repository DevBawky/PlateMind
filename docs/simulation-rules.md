# 타석 시뮬레이션 규칙

## 기본 구조

PlateMind의 타석 시뮬레이션은 0-0 카운트에서 시작해 출루 또는 아웃 결과가 나올 때까지 진행된다. 한 턴은 한 번의 투구를 의미한다.

기본 반복 횟수는 10,000회다.

## 턴 진행

각 턴은 다음 순서로 진행된다.

1. 현재 볼카운트를 확인한다.
2. 현재 카운트에 맞는 구종을 선택한다.
3. 해당 구종과 카운트에서 사용할 수 있는 9개 구역 데이터를 찾는다.
4. 구역별 투구 가능성을 가중치로 사용해 투구 위치를 선택한다.
5. 선택된 구역의 스윙 확률로 타자의 행동을 결정한다.
6. 스윙 또는 지켜봄 결과에 따라 카운트나 타석 결과를 갱신한다.

## 결과 처리

- Ball: 볼이 1개 증가한다.
- Strike: 스트라이크가 1개 증가한다.
- Foul: 2스트라이크 이전에는 스트라이크가 증가하고, 2스트라이크 이후에는 유지된다.
- Hit: 안타로 타석이 종료된다.
- Home Run: 홈런으로 타석이 종료된다.
- Walk: 볼이 4개가 되면 볼넷으로 종료된다.
- Strike Out: 스트라이크가 3개가 되면 삼진으로 종료된다.
- Out: 인플레이 아웃으로 타석이 종료된다.

한 타석의 최대 투구 수는 12개로 제한한다. 12구를 초과하면 현재 상태에 따라 아웃 또는 볼넷으로 종료한다.

## 10,000회 반복 집계

시뮬레이션은 `simulateSinglePlateAppearance`를 10,000회 실행하고 결과를 집계한다.

집계 값은 다음과 같다.

- 출루 확률
- 아웃 확률
- 안타 확률
- 홈런 확률
- 볼넷 확률
- 삼진 확률
- 평균 투구 수
- 평균 압박 지수
- 평균 위험 지수

또한 대표 타석 1개의 턴 로그를 저장해 UI에서 확인할 수 있게 했다.
